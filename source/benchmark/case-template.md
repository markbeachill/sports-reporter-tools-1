# Benchmark Case Template

## Case ID

Example: `MR-001`, `MP-001`, `SR2-001`, `FT-001`

## Working title

Brief descriptive name for the case.

## Article type

Match report, match preview, beginner report, sports feature or other.

## Sport

Name the sport and competition if relevant.

## Source status

State whether the text is original, student-written, commissioned, public-domain, excerpted, linked externally or used under permission.

## Text or excerpt

Paste the test text here if it can be stored in the archive. Otherwise give a short description and a link or storage note.

## Why this case is in the benchmark

Explain what the case is designed to test.

## Expected diagnostic behaviours

List the main things a good tool run should notice. Focus on behaviours rather than exact wording.

Example behaviours:

- identifies the live angle rather than only the final score;
- distinguishes importance from reader attachment;
- notices a strong fact left unconnected to reader care;
- avoids rewriting the article;
- identifies cliché only where it is doing structural damage;
- gives a beginner one reachable move rather than a full expert critique.

## Known traps

List ways the tool might fail on this case.

Examples:

- may invent background match facts;
- may overpraise vivid but inaccurate phrasing;
- may mistake a preview for a report;
- may overload a beginner with too many findings;
- may treat every cliché as equally serious.

## Tools to run

Name the prompt tool or tools to test.

## Notes for scorer

Add any contextual facts needed to judge the run fairly.
